# Ragdoll Sports 布娃娃運動會 — 設計文件 v0.1

> **一句話概念**：兩個軟趴趴的布娃娃打足球，控制混亂 = 笑料爆炸，歪打正著 = 超爽。

---

## 競品分析

| 競品 | 平台 | 核心吸引力 | 弱點 |
|------|------|-----------|------|
| Head Soccer | CG/Mobile | 簡化足球+特殊技，上手快 | 角色太僵硬，缺乏物理笑料 |
| Ragdoll Archers | CG 4.50/5 | 物理回饋爽，爆頭即死感 | 只有射擊，玩法單一 |
| Gang Beasts | Steam | 多人布娃娃格鬥，爆笑 | 太複雜，不適合瀏覽器 |
| Soccer Physics | CG | 2P 物理足球，極簡 | 太簡陋，缺深度 |

**差異化**：保留布娃娃物理的笑料感，但加入真正的運動目標（得分）+ 升級進度 + 多運動項目。

## 六維度設計

### ① 效能

| 設計 | 理由 |
|------|------|
| 每角色 7 個質點（頭/肩/臀/左右膝/左右腳） | 比完整骨骼少一半運算，仍有明顯布娃娃效果 |
| 手臂用程序化渲染（不做物理） | 節省 4 個質點 + 約束，視覺上夠用 |
| 約束迭代 4 次/幀 | 穩定且不燒 CPU |
| 球只有 1 個質點 | 圓形碰撞最簡單 |
| 粒子上限 30 個 | 進球特效不爆記憶體 |
| requestAnimationFrame + deltaTime | 不綁定 60fps，低端手機也順 |

### ② 平衡性

| 設計 | 理由 |
|------|------|
| AI 有 3 段難度（Easy/Normal/Hard） | Easy: 反應慢 0.5s，經常失誤。Hard: 預判球路 |
| 動態難度：連勝 3 場 AI 提升一檔 | 防止玩家碾壓失去興趣 |
| 肌肉力量（站起來的速度）雙方一致 | 公平——AI 也會摔倒 |
| 比賽 90 秒，先得 5 分或時間到高分贏 | 不會拖太久，單局 2 分鐘內結束 |
| 被撞倒後 1.2 秒恢復 | 夠久讓對手趁機進攻，但不至於挫折 |

### ③ UX / 回饋

| 事件 | 視覺回饋 | 音效 |
|------|---------|------|
| 進球 | 螢幕閃白 0.3s + 慢動作 1s + 「GOAL!」大字飛出 + 粒子爆發 | 歡呼音效 |
| 大力踢球 | 球拖尾光跡 + 螢幕微震 | 踢擊音效 |
| 角色被撞飛 | 肢體亂甩 + 星星粒子 | 撞擊音效 |
| 角色站起來 | 身體逐漸收回站姿 | — |
| 計分變化 | 計分板脈動放大 | 叮一聲 |
| 比賽結束 | 贏家跳舞（物理驅動的隨機擺動）| 勝利音樂 |
| 近距離射門未進 | 慢動作 0.5s + 門柱反彈音 | 觀眾嘆息 |

### ⑥ Bug 預防

| 風險 | 預防措施 |
|------|---------|
| 質點穿透地面 | 每幀 clamp Y >= 地面高度 |
| 約束不收斂 → 抖動 | 4 次迭代 + 阻尼 0.99 |
| 球卡在角色體內 | 碰撞後球加反彈推力 |

| AI 卡死 | AI 每 2s 重新評估目標 |
| resize 抖動 | debounce 100ms |

## 核心玩法

### 操控

| 平台 | 移動 | 踢球 | 跳躍 |
|------|------|------|------|
| 桌機 | A/D 或 ←/→ | W 或 ↑ 或 Space | W 或 ↑ |
| 手機 | 左虛擬搖桿 | 右下踢球鈕 | 右上跳躍鈕 |

### 物理系統

```
Ragdoll = 7 質點:
  head(0) - shoulder(1) - hip(2)
                          ├─ leftKnee(3) - leftFoot(4)
                          └─ rightKnee(5) - rightFoot(6)

約束 = 6 條距離約束 + 3 條角度約束（防止劈叉/折頸）

肌肉系統:
  - 正常狀態: 角度約束強力維持站姿
  - 被擊中/摔倒: 肌肉力 → 0（純物理 ragdoll）
  - 1.2 秒後: 肌肉力漸增回來 → 站起來
```

### 球物理

```
Ball = 1 質點 + 半徑
碰撞:
  - 牆/天花板 → 反彈（彈性 0.7）
  - 地面 → 反彈（彈性 0.5）+ 摩擦
  - 角色腳 → 轉移動量 + 踢力加成
  - 角色身體 → 輕微反彈 + 擊倒判定
門 → 球心進入門框 = 得分
```
